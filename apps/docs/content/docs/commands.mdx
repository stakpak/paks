Complete reference for all Paks CLI commands.

## create

Scaffold a new skill from templates.

```bash
paks create <name> [options]
```

### Options

| Option | Description |
| ------ | ----------- |
| `-o, --output <path>` | Output directory (defaults to `./<name>`) |
| `-t, --template <name>` | Template to use: `basic`, `devops`, `coding` (default: `basic`) |
| `--with-scripts` | Include example scripts directory |
| `--with-references` | Include references directory |
| `--with-assets` | Include assets directory |

### Examples

```bash
# Create a basic skill
paks create my-skill

# Create with all optional directories
paks create my-devops-skill --with-scripts --with-references --with-assets

# Create in a specific directory
paks create kubernetes-deploy -o ./skills/k8s
```

## install

Install a skill to your agent's skills directory.

```bash
paks install <source> [options]
```

### Source Formats

| Format | Example | Description |
| ------ | ------- | ----------- |
| Registry | `account/skill` | Install from paks registry |
| Registry with version | `account/skill@1.0.0` | Install specific version |
| Git URL | `https://github.com/user/repo` | Clone from git |
| Git with ref | `https://github.com/user/repo#ref=v1.0.0` | Clone specific tag/branch |
| Git with path | `https://github.com/user/repo#path=skills/my-skill` | Clone subdirectory |
| Local path | `./my-skill` | Copy from local directory |

### Options

| Option | Description |
| ------ | ----------- |
| `-a, --agent <name>` | Target agent (default: configured default) |
| `-d, --dir <path>` | Custom install directory (overrides agent default) |
| `-f, --force` | Force reinstall if already exists |

### Examples

```bash
# Install from registry
paks install stakpak/kubernetes-deploy

# Install specific version
paks install stakpak/kubernetes-deploy@1.2.0

# Install from git
paks install https://github.com/stakpak/skills#path=devops/k8s

# Install for specific agent
paks install stakpak/code-review --agent cursor

# Force reinstall
paks install stakpak/terraform-aws --force
```

## publish

Publish a skill to the registry.

```bash
paks publish <path> [options]
```

### Options

| Option | Description |
| ------ | ----------- |
| `--dry-run` | Show what would happen without publishing |
| `--skip-validation` | Skip SKILL.md validation |
| `-y, --yes` | Skip interactive prompts (auto patch bump) |
| `-t, --tag <tag>` | Use existing git tag instead of creating new |

### Publishing Flow

1. Validates SKILL.md structure
2. Checks git repository status
3. Prompts for version bump (patch/minor/major) or existing tag
4. Creates and pushes git tag
5. Registers with the paks registry

### Examples

```bash
# Dry run to see what would happen
paks publish ./my-skill --dry-run

# Interactive publish (recommended)
paks publish ./my-skill

# Non-interactive with auto patch bump
paks publish ./my-skill --yes

# Publish existing tag
paks publish ./my-skill --tag v1.2.0
```

## validate

Validate skill structure and SKILL.md format.

```bash
paks validate <path> [options]
```

### Options

| Option | Description |
| ------ | ----------- |
| `--strict` | Treat warnings as errors |

### Checks Performed

- ✓ SKILL.md file exists
- ✓ Valid YAML frontmatter
- ✓ Required fields present (`name`, `description`)
- ✓ Field types are correct
- ✓ Name format is valid (lowercase, hyphens)
- ✓ Version follows semver (if present)

### Examples

```bash
# Validate current directory
paks validate .

# Validate specific skill
paks validate ./my-skill

# Strict mode for CI/CD
paks validate ./my-skill --strict
```

| Option | Description |
| ------ | ----------- |
| `--full` | Show full SKILL.md content |

## list

List installed skills.

```bash
paks list [options]
```

### Options

| Option | Description |
| ------ | ----------- |
| `-a, --agent <name>` | Target agent to list skills for |
| `--all` | Show skills from all configured agents |
| `-f, --format <fmt>` | Output format: `table`, `json`, `yaml` |

### Examples

```bash
# List skills for default agent
paks list

# List for specific agent
paks list --agent claude-code

# List all agents' skills
paks list --all

# JSON output for scripting
paks list --format json
```

## remove

Remove an installed skill.

```bash
paks remove <name> [options]
```

### Options

| Option | Description |
| ------ | ----------- |
| `-a, --agent <name>` | Target agent to remove from |
| `--all` | Remove from all agents |
| `-y, --yes` | Skip confirmation prompt |

### Examples

```bash
# Remove skill (with confirmation)
paks remove kubernetes-deploy

# Remove from specific agent
paks remove kubernetes-deploy --agent cursor

# Remove from all agents
paks remove kubernetes-deploy --all --yes
```

## agent

Manage agent configurations.

```bash
paks agent <subcommand>
```

### Subcommands

#### `agent list`

List all configured agents.

```bash
paks agent list
```

#### `agent add`

Add a custom agent configuration.

```bash
paks agent add <name> --dir <skills-directory>
```

#### `agent remove`

Remove a custom agent (built-in agents cannot be removed).

```bash
paks agent remove <name>
```

#### `agent default`

Set the default agent for commands.

```bash
paks agent default <name>
```

#### `agent show`

Show configuration for an agent.

```bash
paks agent show <name>
```

### Built-in Agents

| Agent | Skills Directory |
| ----- | ---------------- |
| `stakpak` | `~/.stakpak/skills` |
| `claude-code` | `~/.claude/skills` |
| `cursor` | `~/.cursor/skills` |
| `vscode` | `~/.vscode/skills` |
| `copilot` | `~/.copilot/skills` |
| `goose` | `~/.goose/skills` |
| `opencode` | `~/.opencode/skills` |

### Examples

```bash
# List all agents
paks agent list

# Add custom agent
paks agent add my-agent --dir ~/.my-agent/skills

# Set default
paks agent default claude-code

# Show agent config
paks agent show stakpak
```
